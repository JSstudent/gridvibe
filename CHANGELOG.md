# Changelog

All notable changes to GridVibe will be documented in this file.

## Unreleased

- **Terminal settings in App Settings:** the App Settings modal gained a **Terminal** section exposing the config that previously required hand-editing `config.json` — terminal font family, font size (6–48), and max sessions (1–16). Values are validated and clamped on save and on config load, persist through `/api/app-config` into `config.json`, and font changes apply to every open terminal window immediately over the existing cross-window app-config contract (max sessions applies to new launches). The modal body also scrolls properly again: the Save/Cancel row is pinned below a scrollable settings area at every window width, so enabling voice input (or the new fields) no longer pushes content under the action buttons. Closes ISSUE-2026-029 and ISSUE-2026-031.
- **Per-agent Auto mode toggle:** agent-mode terminals gained an **Auto mode** checkbox that appears only when the selected built-in agent has a registered auto-mode flag in `agent_registry.json` (`claude --enable-auto-mode`, `codex --full-auto`, `copilot --allow-all-tools`), with help text showing the exact command it will launch. The choice is per terminal and round-trips through launcher drafts, saved sessions, workspace saves, and the restart-restore snapshot, while the stored startup command stays the bare agent key — the flag is appended once at launch time from validated registry metadata, so agent preflight detection keeps working and custom commands are never modified. Closes ISSUE-2026-013.
- **Surgical terminal close:** closing a terminal pane in a split layout is now precise. When geometry allows, exactly one neighbour — the pane sharing the longest border with the closed pane — expands into the freed space instead of every touching pane growing together, and panes outside the expansion keep their manually resized proportions (previously a close reset all panes to uniform widths/heights). Sibling panes in the same group also keep their state across the close: Explorer panes retain their open file/preview, expanded Files tree, and Git sidebar; browser panes keep their URL — where before closing one terminal reset every other pane in the group to a fresh listing/reload. (Browser scroll still resets, since a cross-origin iframe cannot be re-scrolled.) Closes ISSUE-2026-022 and ISSUE-2026-027.
- **Large log previews keep the newest lines:** opening an oversized log in an Explorer pane now retains the **end** of the file instead of the beginning, so current failures and recent activity are visible without an external tail tool. `.log` files keep their last ~1 MiB (trimmed to a clean line and UTF-8 boundary so the preview never starts mid-character or mid-line); every other oversized text file keeps its opening ~1 MiB as before. The file header now reports exactly what you're seeing — `Showing the last <N> of <M>` for logs, `Showing the first <N> of <M>` otherwise — in place of the generic "Preview truncated". Works for both local and SSH/SFTP explorers. Closes ISSUE-2026-020.
- **Markdown preview presentation:** the Explorer Markdown **Preview** got a polished, theme-aware document surface — clearly differentiated `h1`–`h6` (with dividers under the top levels), visible nested list markers, styled links and rules, tables with a header tint and zebra rows, and task-item/image handling. GitHub-style admonition callouts are now supported: a blockquote that starts with `[!NOTE]`, `[!TIP]`, `[!IMPORTANT]`, `[!WARNING]`, or `[!CAUTION]` renders as a coloured callout block with a stroke-style icon and label (emitted server-side *after* sanitization, so the callout classes/icons never widen the HTML sanitizer and callout bodies stay cleaned). A new appearance control in the preview header lets you tune the reading surface: a **Theme** preset (Default / Paper / High contrast) and a **Font** family (System / Serif / Monospace), chosen from an in-page popover. The choice persists locally and applies immediately to every open preview, coexisting with the existing zoom, search highlighting, and light/dark themes. All colours come from theme tokens (no palette literals in the preview rules). Closes ISSUE-2026-017 and ISSUE-2026-030.
- **Explorer tabbed file viewer:** the Explorer main pane is now a persistent read-only *tabbed* viewer. The Files tree drives navigation while the main pane keeps a permanent **Preview** tab (which shows the file you click and starts on an empty "Select a file to view" state) plus closable **pinned** tabs — each file row in the tree gained a small **+** button that opens the file in its own tab, so several reference files stay open while you browse (deduplicated by path, capped at 12; closing the active tab falls back to a neighbour, then the Preview tab). Relative links inside a Markdown **Preview** now resolve against the current file and open as pinned tabs (fragments scroll to the heading; external links open in an isolated new window and never navigate the session page away). Open tabs and the active tab persist in saved sessions and restore on relaunch (out-of-root or missing paths are ignored without failing launch; no file contents are stored). Closes ISSUE-2026-014, ISSUE-2026-016, and ISSUE-2026-015.
- **Explorer rows & Git sidebar polish:** the Files tree and every Git file list now show a compact, extension-aware file-type icon before each name (distinct stroke-style `currentColor` glyphs for code, data, markup, config, shell, SQL, log and more, tinted from theme tokens, with a generic fallback for unknown types). Right-clicking any file row (Files tree or Git sidebar) opens an in-page **Copy path / Copy relative path** menu — keyboard navigable, dismissible on Escape or outside click, and WebView2-safe. Clicking a file under **Changes** or **Staged Changes** now jumps straight to its diff (worktree hunks for unstaged rows, staged hunks for staged rows) instead of opening Source first. And tracked files under **Changes** gained a **Revert** button that, after an in-page confirmation, discards just that file's unstaged edits via `git restore --worktree` (backed by `POST /api/explorer/<id>/git/revert`) while preserving any staged version; untracked and conflicted files are excluded. Closes ISSUE-2026-024, ISSUE-2026-028, ISSUE-2026-023, and ISSUE-2026-018.
- **Terminal search and clickable links:** every terminal pane now supports scrollback search — press `Ctrl+Shift+F` on a focused pane to open a small find overlay (`Enter`/`Shift+Enter` step through matches, `Escape` closes) — and URLs printed in terminal output are clickable, opening in a new tab. Both are powered by the official xterm addons (`xterm-addon-search` 0.13.0, `xterm-addon-web-links` 0.9.0), vendored locally like the rest of the frontend so everything keeps working offline.
- **Broadcast typing:** a new topbar toggle mirrors keystrokes typed into one terminal pane to every other terminal pane in the group (tmux `synchronize-panes` style) — handy for running the same command on N hosts or worktrees. While active, every terminal pane shows an accent border so mirrored input is never a surprise; explorer and browser panes are skipped, and broadcast switches itself off on session-tab switch and after 10 minutes without input.
- **Restore workspace after restart:** GridVibe now snapshots the workspace shape (session groups, pane layout, and per-pane launch settings — never passwords) to a gitignored `runtime_state.json` on every change. After a backend restart, the launcher offers a "Previous workspace found — Restore?" banner that relaunches the same groups through the normal launch path; restored SSH panes authenticate with keys or show the error placeholder's Retry button. Live shell processes and history still cannot survive a restart. Dismissing the banner discards the snapshot; snapshots older than 12 hours are ignored.
- **Explorer file download:** the file viewer in explorer panes gained a download button (`GET /api/explorer/<id>/download`), capped at 100 MB and allowing binary files. In the browser the file downloads directly; in the native desktop window (where WebView2 blocks programmatic downloads) it goes through a native Save dialog instead, and both show a brief confirmation toast. Downloading is a read, so the explorer's read-only contract (which covers filesystem mutations) is unchanged.
- **Configurable SSH host-key verification:** a new `ssh.host_key_policy` setting (also in launcher App Settings under "SSH Host Keys") controls how SSH host keys are checked for terminals, explorers, and agent preflight: `auto-add` (default, today's behaviour — accept new hosts silently, persist to `.known_hosts`, reject changed keys), `known-hosts` (same, but log a warning for every newly accepted host key), or `strict` (reject hosts not already present in the project `.known_hosts` or `~/.ssh/known_hosts`).
- **Reconnect dead panes in place:** errored and disconnected terminal panes now show a "Retry connection" button in their placeholder overlay (backed by a new `POST /api/sessions/<session_id>/reconnect` endpoint) instead of forcing the whole session group to be closed and relaunched. Retrying clears the stale output buffer, resets the pane to "Connecting…", and reuses the normal connect flow, so the other panes in the group keep their state. A shell that exits or a dropped SSH connection now also gets a visible "Disconnected" overlay (file-explorer and browser panes are unaffected — they have no live connection to retry).
- **Closing a session tab now asks first:** clicking a session tab's `×` (or the group close action) shows a confirm dialog naming the group and how many of its terminals are still connected, since live sessions are memory-only and cannot be recovered. Groups with no connected terminals still close with a single click. The dialog uses GridVibe's in-page modal, so it also works in the native window where `confirm()` is blocked.
- **UI polish:** the Launch Workspace button now shows a spinner while launching and keeps its label/arrow markup afterwards; update/settings status messages appear in one place (next to the buttons that trigger them) and clear themselves after 6 seconds instead of showing twice and lingering; the Save Settings tooltip no longer doubles up with the native OS tooltip.
- **Theme sync across windows:** saving a Light/Dark/System appearance change in launcher App Settings now immediately updates every open session window instead of only the launcher. The cross-window app-config contract (BroadcastChannel, storage event, and Socket.IO `app_config_updated`) now carries `appearance.theme` alongside `workspace.surface_mode`, session windows apply both idempotently, and they reconcile the theme against `/api/app-config` on socket reconnect and window focus/pageshow so changes missed while disconnected are recovered. Theme toggles in one window also mirror to other open windows via the shared stored preference. The native Windows title bar stays dark by design (ISSUE-2026-009); closes ISSUE-2026-021.
- **Voice recording indicator:** microphone capture now shows a floating "Recording" pill with live waveform bars and a gentle pulse, centered over the terminals workspace, so push-to-talk and mic-button recordings are visible at a glance. The bars are driven by an `AnalyserNode` on the live capture source (with a deterministic CSS fallback and a `prefers-reduced-motion` static treatment), the overlay is non-blocking, theme-aware, and announced via `role="status"`, and it disappears on every stop path (release, toggle, backend error, permission denial, session switch, teardown). The mic button additionally supports press-and-hold recording (release to stop) alongside the existing click-to-toggle, and rapid key/pointer release during capture startup can no longer leave a stale recording or indicator; closes ISSUE-2026-019.
- **UI consistency:** the launcher and terminals pages now share one design language. A new `web/static/css/tokens.css` defines the shared palette (one accent, one success/danger/warning set, shared radii) that both page stylesheets map onto — the launcher's slate-blue accent and diverging greens were unified on the terminals identity, so moving between the two windows no longer feels like switching apps. The terminal canvas intentionally stays dark on the light theme. All emoji/text-glyph buttons (save/import/delete session, theme toggle, refresh, clear, fullscreen, mic refresh) were replaced with theme-aware stroke-style SVG icons, and the remaining hardcoded colors that ignored the theme (settings-window icon, browser-mode Close button, tooltip arrow, xterm terminal colors) now derive from the CSS variables.
- **Config cleanup (dead-code sweep):** removed three unused `default_config.json` keys — `ssh.default_username` (unused; the per-session username field in saved sessions is the source of truth), `terminal.default_rows`, and `terminal.default_cols` (superseded by the xterm fit-addon). The two useful terminal keys, `terminal.font_size` (default 14) and `terminal.font_family`, are now live: the terminals page reads them and passes them to every xterm instance, so changing either in `config.json` takes effect on next page load.
- **Dead-code removal:** the `/api/sessions/active` HTTP endpoint (no frontend callers, duplicated `/api/sessions` filtering) and the `SessionManager` callback registry (`register_callback` / `_notify_callbacks`) were removed — Socket.IO broadcasts via `_broadcast_session_status` already fill that role at the web layer. The launcher terminal card's inert "⋮ Options" kebab button was also removed (never wired to any action). The native window frame remains permanently dark; the `set_native_theme` JS bridge call no longer passes the resolved app theme as an argument since the Python side intentionally ignores it.

- **Behaviour change:** explicit CLI flags now beat `config.json` — running `python main.py --port 8080` uses port 8080 even when `config.json` says otherwise (previously the config value silently won). Applies to `--host`, `--port`, and `--debug` (which gained a `--no-debug` counterpart) in both `main.py` and `webview_launcher.py`, resolved through one shared helper; both entry points also now start the server through a single shared `run_server`, so startup flags can no longer drift between them.
- **Config cleanup:** `voice_input.vosk_service_url` is now the single source of truth for the Vosk service port — the standalone Vosk service parses the port from the same URL the API dials, so the two can no longer desync. The redundant `vosk_service_port` key was removed from `default_config.json`; if it is still present in `config.json` it is honoured only when the URL carries no port, with a deprecation warning. The Vosk service also now reads the layered config (`config.json` over `default_config.json`) like every other consumer.
- **Security:** `session_status` Socket.IO updates are now emitted only to clients joined to the affected session's room instead of being broadcast to every connected socket, completing the CORS-defaults hardening.
- Fixed the launch button permanently renaming itself from "Launch Workspace →" to "Launch Terminals" after the first launch attempt.
- `utils/cleanup.py` no longer tries to delete the live `logs/gridvibe.log` (it truncates log files in place), no longer treats directories named like file patterns as removable, and its banner now says "GridVibe" instead of the pre-rename project name.
- Log polish: ANSI colour codes are stripped from `logs/gridvibe.log` (console output keeps its colours), and `/api/voice-status` polling no longer clutters the log.
- **Maintainability:** completed the `web/api.py` monolith split (final tranche): the Flask/Socket.IO app objects and the cross-origin write guard now live in `web/app.py`, saved launcher presets and session-config normalization in `web/saved_sessions.py`, the agent registry/detection/preflight and SSH ping probes in `web/agents.py`, the terminal connection registry, output streams, startup sequence, and SSH/local connectors in `web/terminal_io.py`, and the Vosk/faster-whisper voice backends in `web/voice.py`. `web/api.py` is now a ~1,800-line routes-and-handlers module that re-exports every moved name, so `main.py`, the desktop launcher, and the root shims keep working unchanged.
- **Performance:** the launcher and terminals pages no longer ship ~18k lines of inline CSS/JS on every page load. Styles and scripts now live in `web/static/css/` and `web/static/js/` (with a `?v=<version>` cache-buster), so browsers cache them across window opens and reloads; only the small server-rendered constants remain inline. Helpers that were byte-identical between the two pages (theme resolution, directory-resolution rules, connection-mode labels, cross-window sync keys) moved to a single `web/static/js/shared.js` loaded by both pages.
- **Maintainability:** started splitting the `web/api.py` monolith: filesystem locations (`web/paths.py`), Fernet key handling and password encryption (`web/secrets.py`), and the self-update flow (`web/selfupdate.py`) are now standalone modules, with `web.api` re-exporting the public names so existing imports keep working.
- **Maintainability:** second `web/api.py` split tranche: runtime configuration now lives in `web/config.py` (config load/save plus a `RuntimeConfig` object replacing 14 module globals), the shared SSH known-hosts helper in `web/hostkeys.py`, and the entire file-explorer/Git backend (both explorer backends, the single Git helper set, and the SSH connection pool) in `web/explorer.py`. `web.api` re-exports everything, and shrank from ~6,650 to ~4,830 lines.
- **Maintainability:** the JS helpers that had drifted between the launcher and terminals pages (`escHtml`, `joinDirectories`, theme management, the saved-session card renderer) are now reconciled into single implementations in `web/static/js/shared.js`, with small page hooks for page-specific behaviour. One behaviour change: cycling the theme from a terminals window now persists the preference to App Settings, matching the launcher.
- **Maintainability:** decomposed the two largest frontend functions: `buildGrid` (~445 lines) into a pane-instance factory, a card builder, and control-wiring helpers, and `_startVoice` (~308 lines) into microphone-acquisition, capture-pipeline, worklet-message, and teardown helpers — no behaviour change.
- **Maintainability:** the file explorer's local and remote (SSH/SFTP) implementations now share one backend abstraction: every explorer Git helper and route body exists once, parameterised by a local or SFTP backend, removing roughly 900 duplicated lines from `web/api.py`. One payload change: remote Git diff responses now report `byte_count` and `line_count` (matching local diffs) instead of `raw_bytes`.
- **Maintainability:** consolidated the five Git command runners into three with distinct signatures (`_run_repo_git` for self-update, plus one local and one remote explorer runner with a `write=` flag), preventing the runner mix-up that previously caused a production `TypeError`; `POST /api/app-update` now has an end-to-end regression test against a real temporary repository.
- **Reliability:** concurrency hardening across the terminal backend: terminal output is now emitted to Socket.IO clients outside the global connection lock, so one slow or wedged browser connection can no longer stall every other pane's output, input, and resize handling; a session closed while its SSH/local connection was still being established no longer leaks a live SSH client or PTY process; launching a session concurrently with closing another can no longer delete the freshly created session tab (empty groups now get a short cleanup grace period, except when their group is closed explicitly); agent binary preflight probes for different terminals now run in parallel instead of serializing behind a shared cache lock; voice recordings keep using the engine they started with even if the engine is switched in App Settings mid-recording; and short session IDs are regenerated on collision instead of silently overwriting an existing session.
- Fixed workspace `Save Session` and `Save Session as ...` replacing launcher repository roots and per-terminal directories with transient File Explorer navigation paths; workspace saves now preserve launcher connection/directory setup while updating pane modes (including registered agents started interactively with commands such as `codex` or `claude`), counts, ordering, and resize geometry.
- Saved Explorer panes now restore whether the File Tree and Git sidebars were open when the workspace session was saved.
- Agent panes now return to persisted terminal mode after an explicit CLI exit, clearing the stale agent selection and startup command before the workspace is saved.
- Added an icon-only refresh control to the file explorer toolbar, immediately before the parent-directory button, using the existing scroll-preserving directory/file and sidebar refresh flow.
- Fixed session tab switching moving terminal views to the top; cached panes now restore exact xterm viewport positions after redraw, keep bottom-following panes at the live buffer bottom, and ignore stale delayed restores from rapid tab switches.
- **Performance:** remote file explorer requests now reuse a pooled SSH connection per session instead of opening a fresh TCP + SSH + SFTP handshake on every directory listing, file preview, or Git action, removing roughly 300–1000 ms of latency per click. Idle pooled connections close after 60 seconds and are evicted when the session closes.
- **Performance:** terminal replay buffers now store output chunks in a deque with an exact 50 KB rolling trim instead of re-copying the whole buffer on every chunk, shortening lock hold times for busy panes.
- **Performance:** the terminals page no longer polls session state every 3 seconds. The backend pushes a `session_groups_updated` Socket.IO event on launch, split, close, and tab reorder, and the page refreshes on push; a 15-second reconciliation poll remains only while the Socket.IO connection is down.
- **Reliability:** the terminals page no longer loads xterm.js, the fit addon, or the Socket.IO client from `cdn.jsdelivr.net`. Pinned copies (xterm 5.3.0, xterm-addon-fit 0.8.0, socket.io-client 4.7.2) are vendored under `web/static/vendor/` and served locally, so terminals work fully offline and no third-party CDN script runs with access to the terminal.
- **Performance:** idle SSH panes no longer poll their channel 20 times per second. The output stream now blocks on `recv` with a 0.5-second timeout, cutting idle wakeups and global-lock churn.
- **Performance:** the rare no-PTY fallback path for local shells now reads output in buffered chunks instead of one byte (and one Socket.IO message) per character.
- **Security (breaking for reverse-proxy setups):** Socket.IO CORS now defaults to same-origin (`http://127.0.0.1:<port>` / `http://localhost:<port>`) instead of `*`. If GridVibe is served from another origin (for example behind a reverse proxy), set `security.cors_origins` in `config.json` explicitly; `["*"]` restores the old behaviour.
- **Security:** state-changing HTTP requests (`POST`/`DELETE`/`PUT`/`PATCH`) with a cross-origin `Origin` header are now rejected with `403`, blocking drive-by requests from hostile web pages against the local API. Origins listed in `security.cors_origins` remain allowed.
- **Security:** SSH host keys are now persisted to a project-local `.known_hosts` file (gitignored). First connections still auto-accept the key, but a changed host key on a later connection is rejected instead of being silently accepted.
- Added SSH keepalive for terminal sessions: the existing `ssh.keepalive_interval` config key (default 60 seconds, `0` disables) is now applied to SSH connections so idle panes behind NAT/firewalls stay alive.
- Fixed the launcher's `💾 Save Session` button doing nothing in the native desktop window: the session-name prompt is now an in-page modal instead of `window.prompt`, which WebView2 blocks.
- Fixed stored SSH passwords being sent as raw ciphertext when the encryption key changed; undecryptable passwords are now ignored with a warning in the log.
- Fixed PowerShell startup-directory quoting so paths with spaces (for example under `C:\Program Files`) change directory correctly.
- Fixed the terminal selection highlight colour, changing the whisper model/device in App Settings now taking effect without a restart, a duplicate-tab-order race when launching sessions concurrently, and unstyled info/warning launcher status messages.
- Reduced log noise: normal pane closures on Windows are now logged at debug level instead of ERROR.
- Added basic Git staging to the file explorer Git sidebar: unstaged change rows show a `+` stage button and staged rows show a `-` unstage button, with a separate `Staged Changes` section above the working `Changes` list. Available for Local Repo and SSH explorer panes.
- Added a commit message box with a `Commit` button that commits the staged changes, and a `Publish branch` button that pushes the current branch (setting the upstream to `origin` when none exists). Mutating Git commands never prompt for credentials, so publishing fails fast instead of hanging when authentication is required.
- Added a Windows console launch prompt with Desktop, Browser, and Quit choices; Desktop uses native-first startup with browser fallback, while Browser skips optional desktop dependency setup.
- Added a token-protected Close button beside the Launcher Setup Session and Mode badges when running in explicit Browser mode, allowing the browser UI to stop the GridVibe Python process.
- Added `GridVibe.sh` for Ubuntu/Linux startup with a terminal mode prompt, plus explicit `webview_launcher.py --mode browser|native|auto` launch modes.
- Updated dependency floors and the Windows launcher dependency bootstrap so existing virtualenvs upgrade, verify native imports, and repair stale compiled wheels before startup.
- Enabled read-only File Explorer panes for SSH sessions using SFTP with root-bound remote path validation and terminal/explorer mode switching.
- Added read-only Git awareness to Local Repo and SSH file explorer panes, including branch/dirty summaries, per-entry status badges, directory dirty markers, and bounded internal old/new diff previews with added/removed line highlighting.
- Added a resizable Git sidebar for Local Repo and SSH file explorer panes, showing uncommitted changed files plus a collapsible commit graph with per-commit changed files, folder navigation, and read-only historical diffs from commit file rows.
- Added a `Tree` file tree sidebar to Local Repo and SSH file explorer panes with lazy per-directory loading, in-place expand/collapse, Git status badges, folder jump buttons, and automatic reveal of the current directory or open file. The tree and Git sidebars can be open at the same time, stacked in one shared sidebar with a draggable horizontal divider.
- Added numbered session tabs with `Alt+1` through `Alt+9` switching based on the current tab order, while ignoring editable fields.
- Added drag-resizable terminal pane dividers with xterm refits, backend PTY resize propagation, session-local resize weight caching, and minimum pane-size guards.
- Added per-pane `-`/`+` font-size zoom controls to the read-only file editor view, covering source and Markdown preview panels.
- Added source line numbers to read-only explorer file previews and Markdown source-gutter chevrons for collapsing heading sections.
- Added a global App Settings option for normal vs. max session-window surface mode while keeping the in-session surface toggle available.
- Improved Git sidebar readability by colouring each commit graph lane separately, so mainline and branch commits are distinguishable, and gave the folder jump buttons in the Git and Tree sidebars a shared light-blue style that adapts to the light and dark explorer themes.
- Moved the file explorer `Tree` and `Git` sidebar toggles, along with the Git branch summary, to the left of the directory path, and replaced the toggle labels with file-hierarchy and Git-branch icons.

## 1.1.0 - 2026-06-28

- Improved Linux pywebview startup by requesting the Qt backend directly, keeping QtWebEngine GPU fallback flags opt-in, and preventing terminal job-control suspension when launching the native window.
- Reduced repeated agent CLI preflight timeouts by checking local POSIX PATH before interactive shells and caching recent identical local detections.
- Added Local Repo file explorer panes with safe root-bound directory navigation, file metadata, manual refresh, light/dark explorer themes, and terminal-to-explorer pane switching.
- Added Linux desktop dependency coverage for pywebview's Qt backend and a clearer browser fallback when no GTK/Qt backend can be loaded.
- Added read-only text file previews, Markdown source/rendered preview support, sanitized Markdown HTML, code-language hints, binary-file rejection, and size-limited explorer responses.
- Added split-pane support for terminal sessions, including API coverage for appending cloned sessions to an existing group while enforcing session limits.
- Improved saved launcher preset handling and launcher settings so voice capture preferences live in App Settings instead of per-terminal voice panels.
- Improved terminal workspace behavior around explorer refreshes, pane mode switches, session deletion, replay-buffer sanitization, and local stream shutdown handling.
- Added Markdown and HTML sanitization runtime dependencies for explorer previews.
- Updated screenshots, README workspace documentation, and the voice guideline to reflect the current launcher, explorer, and microphone settings flow.
- Expanded API and session-manager tests for explorer path safety, file previews, pane switching, split sessions, saved sessions, voice preferences, and SSH/local-session error handling.
- Updated developer automation so `make check`, `make lint`, `make test`, and `make fix` bootstrap development dependencies through `.venv`.

## 0.1.0 - Initial public release

- Browser-first multi-session terminal workspace.
- SSH and local shell session launch modes.
- Saved presets with encrypted SSH passwords.
- Session groups with tab ordering and replay buffers.
- Optional native desktop window support through pywebview.
- Optional offline voice input through Vosk or faster-whisper.
